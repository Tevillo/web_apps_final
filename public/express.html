Express intro

The following features are implemented by this server:

pathFunctionality
hello.html A simple implementation of a hello document. There is both a hello.html in the static folder of Express, but there is also a path definied in the app. Whichever is called first will handle this request.
dog.jpg Simply showcasing express as a file server.
hellodog.html A demonstration that you can just use relative paths for resources in the public folder.
heyho A route demonstration. Consider this a "Hello World" of routes.
params?monkey=me&audience=you Showcasing express' ability to automatically parse the parameters. This simple route just iterates over all of the parameters, and lists them in an unordered list.
params Same path called without any actual parameters.
postDemo.html Demonstration of handling post information. postDemo.html is the page with the webform. It is then passed on to /postDemo, which is a route that processes the data.
postDemo Calling postDemo (the route to process the form data) directly will result in an error explaining it cannot be GET.
people/Elon Musk Demonstration of the path marameters. The people path will only match if a parameter is given to it.
people Calling people without a parameter will result in a 404 error as no routes match.
